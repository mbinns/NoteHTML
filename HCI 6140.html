Cognative Walkthrough

Have a user of an expert walkthrough the process similar to a thinkaloud but it is not the same thing.

Difference between thinkaloud and cognative

  • Thinkaloud
    • Describe your feelings about the interface
    • Why they are preforming the actions they are
    • Novice users
  • Cognative
    • Experts are doing the test and you are not describing feelings
    • Trying to get into the mindset of the novice users

Similarites between thinkaloud and cognative

  • Trying to identify where users are going to get off task

Definition

  • http://www.usability.gov/what-and-why/glossary/cognitive-walkthrough.html
  • Inspection method
    • Have experts inspect a particular interface or prototype
    • Can be preformed in the early stages of design
      • Most bang for your buck
  • We ask specific questions as we step through the interface
    • Will users be trying to produce whatever effect a particular action has
    • Will users notice a correct action is visible
    • Will the user associate the correct action with the effect to be achived
    • Will users understand feedback after action

Cognative Walkthroughs are really well suited to first time users doing exploaratory learnng with out any formal training. You are typing to imagine what a novice user might encounter when using a system.

Requirements

  • Who will be the users of the system
  • What tasks will be analyzed
  • What is the correct action sequence for each task
    • What is the path that the users must take to complete the task at hand
  • Decription or implementation of the interface
    • Doesn't have to be something that is up and running
      • Could be a prototype or a very good description of the system
  • Usability experts (1-4)
    • 3 - 4 is the ideal number

Assumptions

  • A user may have some similar cultural background so that they would be familiar with a particular metaphore
  • Similar Physical abilities
  • The user should have a Goal defined
    • Vague goals do not lend themselves well to a cognative walkthrough
  • The user will explore
    • Older users maybe afraid to that they are going to break the system
  • The users are going to choose something that seems reasonable
  • The user is able to interpret the response from the system correctly

Process

  • Finding some collegues to participate in it with you
  • Define some tasks
  • Define the sequence of actions that ideally would allow users to accomplesh a particular goal
  • Take notes and keep track of things noted
  • Analyise data

Methodology

As a walkthrough is being conducted record your assumptions

  • Assumptions about what would cause problems and why they are recorded- why users face difficulties
  • Notes about side issues and design changes
  • Summary of results

Design is revised to fix the problems presented

User Support

  • Help:
    • Problem oriented and specific
  • Documentation:
    • System oriented and general
  • Essential:
    • This is not a replacement for bad design
  • Simple system:
    • Walk up and use it
    • Name some:
      • ATM
      • Single button systems
      • Parking meters
      • Fire Alarms
      • Airport kiosk
  • Most systems with reach features (even well-designed ones) require help and documentation

Manual

  • Cons:
    • Dense
    • Can be lost
    • Hard to find what you are looking for
    • Users just don't read them
  • Designing the manual to be easily accessable:
    • Include elements of the manual on the devices
    • Make it available online
  • Requirements:
    • Availability:
      • Anytime the user is operating the system
    • Accuracy and completeness:
      • Tricky with changing versions
      • Cover all aspects of the system
    • Consistency:
      • Across different sections
      • Between online and paper documentation
      • In terms of terminiology, content and style
    • Robustness:
      • Predictable and free of errors
    • Flexability:
      • Appropriate for novices through experts:
        • Possibly have an expanded section with more detaill
    • Unobtrusive:
      • Shouldn't distract from or interfere with normal work flow
        • Having to pull out a manual from a box or something
      • Man pages are very good at this
