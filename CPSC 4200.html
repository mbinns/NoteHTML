8/23/16

  • The HB gary attack
    • Used SQL injection to gain access to the CMS server
    • Then they used a rainbow table to crack the password hashes:
      • A Rainbow table is similar to a hybrid dictionary attack in that it takes a word list and runs through all the words but then it starts changing letters in the word to common substitutions (I.E. a 0 for an o) this table is pregenerated to take some of the computational load off the system while cracking
    • After recovering a password they then used it to log into the webserver and used an escalation exploit to gain administrator access
    • The pivioted to the email server and used the emails to social engineer access to other machines
  • Lessons Learned from HBGary Hack:
    • Use off the shelf software
    • Patching!
    • Store passwords with stronger security
      • They used MD5 with no iterative hashing or salting
    • Force complex passwords
    • Don't reuse passwords
    • Delete sensitive e-mails
      • An email containing the root password for the rootkit.com site
    • Educate against social engineering

  • Defining Security:
    • The security of a system, application, or protocol is always relative to a few things:
      • A set of desired properties
      • An adversary with specific capabilities
  • CIA
  • Confidentiality:
    • Avoidance of unauthorized disclosure of information
    • Protection of data
    • Providing access to those who are allowed to see it
    • Disallowing access to everyone else
    • Tools for confidentiality:
      • Encryption:
        • Public and private keys so what when data is transmitted it is unreadable
      • ACL/Authorization:
        • Rules and policies that limit access to configential info
        • Access can be determinied by many factorsr:
          • What you are:
            • Biometrics
            • Roles
          • What you know:
            • Username
            • Password
          • What you have:
            • Smartcard
            • Keys
      • Physical security:
        • Locks
        • Doors
        • Windowless rooms
        • Sounds dampening
        • Faraday Cages
        • Air gap
  • Integrity:
    • The data is still in its original form and not been altered
    • Tools for intergrity:
      • Backups:
      • Checksum:
        • The computation of a function that maps the contents of a file to a cumerical value, A checksum function depends on the entire contents of a file and is designed in a way that even a small change to the input is likely to result in a different outut from the hash funciton
      • Data corecting codes:
        • Method for storing where small changes can be seen and fixed
  • Availability:
    • The property of data where it is ready to be used in a timely manner
    • Tools for availability:
      • Computational redundancies:
        • Computers and storage devices that serve as fallbacks in the case of failures
      • Physical protections:
        • Infranstructure meant to keep information avaiable even in the even of physical challenges (Natrual disasters)
  • Other security concepts:
    • A.A.A:
      • Authenticity:
        • Ability to determine the assureance properties are genuine
        • Authenticity tools:
          • digital signatures:
            • Systems commit to the authenticity of thier documents with nonrepudiation (no take backsies)
      • Assurance:
        • Refers to how trust is provided and managed in computer systems
        • Trust management:
          • Policies:
            • Behavioral expectations that people or systems must follow
          • Permissions:
            • Describe behaviors allowed by agents that interact with a system
          • Protections:
            • mechanisms put in place to enforce permissions and policies
      • Anonymity:
        • Certian records cannot be attributed to any individual
        • Toos for anonymity:
          • Aggregation of data:
            • Mixing of all data so it is harder to pin down on one user
          • Mixing:
            • Interwinning of transactions, information, or communication in a way that cannot be traced to any individual
  • Threats and Attacks:
    • Eavesdripping:
      • Intercepting of information on its way to its destination
      • Packet sniffing
      • An attack on confidentiality
    • Alteration:
      • MITM Attacks
        • Where an attacker has a network stream sent to them where it can be copied, modified, and then retransmitted
      • Attack on Confidentiality, Integrity
    • DOS:
      • Itteruption in a service by sending lots of traffic/data to a server
      • Attack on avaibality
    • Masquerading(Impersinating):
      • Social engineering attacks like phishing and spoofing
      • An attack on authenticity/confidentiality/anonymity
    • Repudiation:
      • Denail of a commitment or data receipt
      • Attempting to back out in the middle of a protocol that requires the differnet parties to provide receipts ack that they have gotten data
      • Attack on assurance
    • Correlation and traceback:
      • The integration of multiple data sources and information flows to figureout the source of information
      • An attack on anonymity
  • Authentication v Authorization:
    • Authentication:
      • Who or what can access a system
      • User name and password
    • Authorization:
      • Are you allowed to do that
      • ACP (Access control policies)
      • Restrictions on actions of authenticated users
